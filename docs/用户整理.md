# 用户自己整理的

### nodeCompat 字段的背景

我们系统有两套标准库，一套是我们的标准，一套是node的标准。我们日后要用的是我们的标准库，现在的node标准库只做为能够运行node包使用，懂了么？
为了实现这个逻辑，我们在原来的oolong.json =》cha.json 中，增加了一个 nodeCompat 字段，当我们这个项目，运行的是node项目的时候，项目需要增加 nodeCompat 才能正常将 import "path" from "path" 解析到 import path from node:path。
如果我们是新项目，用的是我们的标准库，不加该 nodeCompat 字段。为什么呢？因为默认的标准库就是 import "path" from "path" 就是我们的默认标准库的短写 ，等于 import path from "@std/path"。
正因为我们默认的 alias 是我们自己的标准库，我们才需要用这个字段来让运行时，知道是该找哪个模块。

当然，如果新项目，里面使用 import path from node:path，那就不能加该 nodeCompat 字段，为什么？因为这只是我们临时借调node模块，而且是全量名，模块加载器可以正常加载。
